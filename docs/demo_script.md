# Metoy 科学小导师演示脚本

这份脚本用于录屏、答辩或报告截图。建议按顺序演示，每个问题等待回答后截图。

## 演示目标

展示系统不是普通聊天框，而是一个带本地知识库、教具工具、上下文记忆和开发者可解释界面的 LLM Agent。

## 演示 1：知识点讲解

学生输入：

```text
彩虹是怎么形成的？
```

预期展示点：

- Agent 应该先讲知识点，不直接设计实验。
- 回答适合学生理解。
- 最后可以自然询问是否要用实验验证。

截图建议：

- 学生聊天界面完整回答。

## 演示 2：标准教具目录查询

学生输入：

```text
你们有哪些教具？
```

预期展示点：

- Agent 应该调用教具目录工具。
- 教具名称来自本地说明书。
- 不应该编造不存在的教具。

截图建议：

- 教具清单回答。
- 如果出现可点击卡片，也截图卡片区域。

## 演示 3：基于教具的实验指导

学生输入：

```text
我想用平面镜成像演示仪学习平面镜成像，请带我一步步做实验。
```

预期展示点：

- Agent 应该给出实验目标、材料、步骤、观察点和安全提醒。
- 回答应绑定本地说明书或相关资料。

截图建议：

- 学生端回答。
- 开发者控制台中的 RAG 依据。

## 演示 4：上下文追问

学生输入：

```text
下一步呢？
```

或：

```text
怎么验证呢？
```

预期展示点：

- Agent 应该接着上一轮实验，不重新问 A/B/C。
- 不能跳到更早的无关话题。

截图建议：

- 连续两轮对话截图，突出上下文记忆。

## 演示 5：生活化科学问题

学生输入：

```text
怎么验证铁和棉花一样重？
```

预期展示点：

- Agent 应该围绕质量、重量、天平或称量实验解释。
- 适合学生理解。
- 可以引导动手验证。

截图建议：

- 学生端回答。

## 演示 6：开发者控制台

操作：

1. 左侧选择“开发者控制台”。
2. 输入一个实验设计需求。
3. 点击运行 Agent。

示例输入：

```text
我想让学生通过一个益智玩具理解平面镜成像的特点，并能自己动手验证。
```

预期展示点：

- `Agent 架构` 展示流程。
- `Agent 轨迹` 展示每一步。
- `RAG 依据` 展示命中文档。
- `质量校验` 展示回答是否通过基础检查。

截图建议：

- Agent 架构页。
- RAG 依据页。
- 质量校验页。

## 演示 7：商店入口

操作：

1. 点击左侧栏购物车图标。
2. 展示教具商店页面。
3. 点击返回聊天。

预期展示点：

- Agent 不只是问答 demo，也可以和教具产品页面结合。

## 报告中可以放的样例表格

| 测试问题 | 目标能力 | 预期结果 |
| --- | --- | --- |
| 彩虹是怎么形成的？ | 知识点讲解 | 先讲概念，不直接做实验 |
| 你们有哪些教具？ | 工具调用 | 从本地说明书列教具 |
| 用平面镜成像演示仪带我做实验 | RAG + 实验设计 | 给出步骤和安全提醒 |
| 下一步呢？ | 上下文记忆 | 接着上一轮继续 |
| 怎么验证铁和棉花一样重？ | 生活化实验 | 给出称量验证思路 |
| 光的偏振是什么？ | 概念解释 | 用学生能懂的语言解释 |

## 答辩时一句话介绍

我的项目是一个面向科学教具学习的 EduToy Agent。它结合 GLM、本地 RAG、教具目录工具和上下文记忆，让学生可以通过聊天学习科学知识，并基于真实教具说明书获得实验指导；开发者界面还能展示 Agent 的检索依据和运行轨迹。
